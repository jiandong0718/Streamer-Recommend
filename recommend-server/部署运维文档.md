# 主播推荐系统部署运维文档

## 1. 环境准备

### 1.1 基础环境要求

**硬件要求**
- CPU: 8核以上
- 内存: 16GB以上
- 磁盘: SSD 500GB以上
- 网络: 千兆网卡

**软件环境**
- 操作系统: Ubuntu 20.04 LTS / CentOS 8
- Java: OpenJDK 17
- Maven: 3.8+
- Docker: 20.10+
- Docker Compose: 2.0+

### 1.2 中间件环境

**数据库**
- MySQL 8.0
- 配置要求: 8GB内存，SSD存储

**缓存**
- Redis 7.0
- 配置要求: 4GB内存

**消息队列**
- RabbitMQ 3.11 (可选)

**配置中心**
- Nacos 2.2.0

**监控**
- Prometheus 2.40
- Grafana 9.3
- Alertmanager 0.25

## 2. 环境配置

### 2.1 MySQL配置

**安装MySQL**
```bash
# Ubuntu
sudo apt update
sudo apt install mysql-server-8.0

# CentOS
sudo yum install mysql-server
```

**配置文件 (/etc/mysql/mysql.conf.d/mysqld.cnf)**
```ini
[mysqld]
# 基础配置
port = 3306
bind-address = 0.0.0.0
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci

# 性能优化
innodb_buffer_pool_size = 6G
innodb_log_file_size = 512M
innodb_flush_log_at_trx_commit = 2
innodb_flush_method = O_DIRECT

# 连接配置
max_connections = 1000
wait_timeout = 600
interactive_timeout = 600

# 日志配置
log-error = /var/log/mysql/error.log
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 2
```

**数据库初始化**
```sql
-- 创建数据库
CREATE DATABASE recommend_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建用户
CREATE USER 'recommend_user'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON recommend_db.* TO 'recommend_user'@'%';
FLUSH PRIVILEGES;
```

### 2.2 Redis配置

**安装Redis**
```bash
# Ubuntu
sudo apt install redis-server

# CentOS
sudo yum install redis
```

**配置文件 (/etc/redis/redis.conf)**
```conf
# 网络配置
bind 0.0.0.0
port 6379
protected-mode yes
requirepass your_redis_password

# 内存配置
maxmemory 4gb
maxmemory-policy allkeys-lru

# 持久化配置
save 900 1
save 300 10
save 60 10000
rdbcompression yes
rdbchecksum yes

# 日志配置
loglevel notice
logfile /var/log/redis/redis-server.log
```

**Docker方式部署**
```bash
# 下载配置

# 启动Nacos
```

## 3. 应用部署

### 3.1 构建部署

**Maven构建**
```bash
# 清理并打包
mvn clean package -DskipTests

# 构建Docker镜像
docker build -t recommend-server:latest .
```

**Dockerfile**
```dockerfile
FROM openjdk:17-jre-slim

LABEL maintainer="recommend-team"

# 创建应用目录
RUN mkdir -p /app/logs

# 复制应用文件
COPY target/recommend-server-*.jar /app/app.jar

# 设置工作目录
WORKDIR /app

# 暴露端口
EXPOSE 8080

# JVM参数配置
ENV JAVA_OPTS="-Xms2g -Xmx4g -XX:+UseG1GC -XX:+PrintGCDetails -XX:+PrintGCTimeStamps -Xloggc:logs/gc.log"

# 启动应用
ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar app.jar"]
```

**recommend_rules.yml**
```yaml
groups:
- name: recommend.rules
  rules:
  - alert: HighErrorRate
    expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.1
    for: 5m
    labels:
      severity: critical
    annotations:
      summary: "High error rate detected"
      description: "Error rate is {{ $value }} errors per second"

  - alert: HighResponseTime
    expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 1
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "High response time detected"
      description: "95th percentile response time is {{ $value }} seconds"

  - alert: LowCacheHitRate
    expr: recommend_cache_hit_rate < 0.8
    for: 10m
    labels:
      severity: warning
    annotations:
      summary: "Low cache hit rate"
      description: "Cache hit rate is {{ $value }}"
```

## 5. 运维脚本

### 5.1 部署脚本

**deploy.sh**
```bash
#!/bin/bash

set -e

# 配置变量
APP_NAME="recommend-server"
VERSION="${1:-latest}"
ENVIRONMENT="${2:-prod}"

echo "开始部署 $APP_NAME:$VERSION 到 $ENVIRONMENT 环境"

# 停止旧版本
echo "停止旧版本..."
docker-compose down

# 拉取新镜像
echo "拉取新镜像..."
docker pull $APP_NAME:$VERSION

# 更新配置
echo "更新配置..."
cp config/application-$ENVIRONMENT.yml ./

# 启动新版本
echo "启动新版本..."
docker-compose up -d

# 健康检查
echo "等待服务启动..."
sleep 30

# 检查服务状态
for i in {1..10}; do
  if curl -f http://localhost:8080/actuator/health; then
    echo "服务启动成功"
    exit 0
  fi
  echo "等待服务启动... ($i/10)"
  sleep 10
done

echo "服务启动失败"
exit 1
```

### 5.2 备份脚本

**backup.sh**
```bash
#!/bin/bash

set -e

# 配置
BACKUP_DIR="/data/backup"
DATE=$(date +%Y%m%d_%H%M%S)
MYSQL_CONTAINER="mysql"
REDIS_CONTAINER="redis"

mkdir -p $BACKUP_DIR

echo "开始备份数据库..."

# 备份MySQL
docker exec $MYSQL_CONTAINER mysqldump -u root -p$MYSQL_ROOT_PASSWORD recommend_db > $BACKUP_DIR/mysql_$DATE.sql

# 备份Redis
docker exec $REDIS_CONTAINER redis-cli --rdb $BACKUP_DIR/redis_$DATE.rdb

# 压缩备份文件
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz $BACKUP_DIR/*_$DATE.*

# 清理旧备份（保留7天）
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +7 -delete

echo "备份完成: backup_$DATE.tar.gz"
```

### 5.3 监控脚本

**health_check.sh**
```bash
#!/bin/bash

# 检查服务健康状态
check_service() {
    local service_name=$1
    local health_url=$2
    
    echo "检查 $service_name 健康状态..."
    
    response=$(curl -s -o /dev/null -w "%{http_code}" $health_url)
    
    if [ $response -eq 200 ]; then
        echo "✓ $service_name 正常"
        return 0
    else
        echo "✗ $service_name 异常 (HTTP: $response)"
        return 1
    fi
}

# 检查应用服务
check_service "推荐服务" "http://localhost:8080/actuator/health"

# 检查数据库连接
check_service "数据库检查" "http://localhost:8080/actuator/health/db"

# 检查Redis连接
check_service "Redis检查" "http://localhost:8080/actuator/health/redis"

echo "健康检查完成"
```

## 7. 安全配置

### 7.1 网络安全

**防火墙配置**
```bash
# 开放必要端口
ufw allow 22/tcp    # SSH
ufw allow 8080/tcp  # 应用端口
ufw allow 3306/tcp  # MySQL (仅内网)
ufw allow 6379/tcp  # Redis (仅内网)

# 启用防火墙
ufw enable
```

**SSL/TLS配置**
```nginx
server {
    listen 443 ssl http2;
    server_name api.recommend.com;
    
    ssl_certificate /etc/ssl/certs/recommend.crt;
    ssl_certificate_key /etc/ssl/private/recommend.key;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-GCM-SHA384;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 8. 灾难恢复

### 8.1 备份策略

**数据备份**
- 全量备份：每日凌晨2点
- 增量备份：每小时一次
- 异地备份：每日同步到异地机房

**配置备份**
- 配置文件版本控制
- 镜像备份到多个仓库
- 部署脚本版本管理

### 8.2 恢复流程

**数据库恢复**
```bash
# 恢复MySQL
mysql -u root -p recommend_db < backup/mysql_20241220_020000.sql

# 恢复Redis
redis-cli --rdb backup/redis_20241220_020000.rdb
```

**服务恢复**
```bash
# 快速回滚到上一版本
docker-compose down
docker tag recommend-server:current recommend-server:rollback
docker tag recommend-server:previous recommend-server:current
docker-compose up -d
```
